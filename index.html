The three contenders:

  • Ruby on Rails

Ruby on Rails

  • First released in 2005 by DHH (a Dane!)
  • Set a new standard for web frameworks
  • Very well documented with strong community
  • Uses (as in the name) Ruby, a lesser-known language

Laravel

  • Uses PHP, the most widely used language for web back-ends
  • Newer (released 2011), but arguably the best PHP framework
  • Provides a lot of functionality out of the box
  • Good community and great documentation

ASP.NET MVC

  • The veteran of the pack, Released in 2002 by Microsoft
  • A core part of the extensively-documented .NET framework
  • Well-established, official support from Microsoft and very large community
  • Recently made open-source as part of a larger effort by Microsoft

So...what is a web framework?

Hint: it's just code, there's nothing magical going on under the hood!

The super-duper basics:

  • Web applications have a front-end and a back-end.
  • The front-end runs in every user's web browser, while the back-end will be on a web server.
  • All the important logic and things like authentication and database access should take place on the back-end.
  • The front-end sends requests to the back-end, which sends responses back.
  • A front-end is only ever written in HTML, CSS & JS.
  • A back-end is written in any language (but usually something like PHP or Java or C#).

Frameworks: Your problems have already been solved!

If you want to build a database-backed application on the web, why reinvent the wheel?

Frameworks give you:

  • A well-paved development path
  • Architecture, testing and tooling
  • Authentication and security (NEVER do these yourself!!!!!!)
  • Documentation and support
  • Community
  • Opinions gained from collective experience

A web framework takes care of wiring together all the nitty-gritty stuff common to most web applications.

This frees you to work on the things that make your application unique.

What frameworks are not...

  • A Content Management System (CMS)
    • WordPress, Squarespace, Wikis, Drupal, etc.
    • No drag-and-drop interfaces.
  • An out-of-the-box solution
    • You still have to code out your application.
  • Perfect
    • Even the most well-established frameworks have bugs and flaws. They are made by people, after all...

Architecture

This will only sting a little...

Similar programs follow similar patterns

Most web applications have to take care of:

  1. Showing something to the user
  2. Processing user input
  3. Operations on data

MVC

MVC separates these three actions into:

  • Model
    • What the application does, including data
  • View
    • What the user sees and interacts with
  • Controller
    • Acts on input, glues View and Model together

N-Tier

N-tier applications are split up into layers:

  • Presentation tier
    • Interacts with user
  • Middle Tier
    • All sorts of juicy program logic
  • Data tier
    • Database and database connection

Roundup on architecture

With experience you'll get used to seeing MVC application frameworks, they're everywhere

Remember: "Model, View, Glue"

N-Tier is more common in ground-up enterprise applications

Tooling

The command line is your friend...

  • Rails uses RubyGems to provide extensions
  • Laravel uses Composer
  • In ASP.NET, Visual Studio provides everything (now via NuGet)

Text Editors

  • Rails tends to favor text editors like Sublime and NPP
    • The magic happens on the command line!
    • IDEs like RubyMine are also available
  • Laravel can go either way
  • ASP.NET (almost) requires the use of Visual Studio

Choosing a framework

Community

Community means:

  • Widespread Documentation
  • Quality tutorials
  • Local meetups
  • Informal support
  • Open source tooling

Community, compared

  • Rails is famous for its now-well established community. It suffers from a monoculture, so if you're not on the right platform, you might struggle.
  • Laravel is newer, but has a growing community. You will most likely to find support and tooling and documentation for whatever you're working on.
  • ASP.NET has been around for a long time and is a Microsoft flagship product. The community is very large, but almost entirely corporate professional. Open source .NET projects are less common.
